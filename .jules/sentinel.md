## 2024-05-24 - [CRITICAL] Prevent Global Cleartext Traffic
**Vulnerability:** Android application explicitly allowed all HTTP traffic globally via `android:usesCleartextTraffic="true"`.
**Learning:** This exposes the app to MITM attacks for all network communication.
**Prevention:** Use a `network_security_config.xml` to globally enforce HTTPS (`cleartextTrafficPermitted="false"`) while maintaining explicit exceptions for local development domains (e.g., `10.0.2.2`, `localhost`).
## 2026-04-22 - [HIGH] Missing Rate Limiting on API Endpoints\n**Vulnerability:** The Express backend lacked rate limiting, exposing the API and LLM endpoints to DoS and abuse.\n**Learning:** In a production Express environment, sensitive endpoints (especially those interacting with paid/rate-limited external AI APIs) must be protected with rate limiting. This also requires correctly configuring `app.set('trust proxy', 1)` when behind reverse proxies like Vercel to avoid global rate limiting.\n**Prevention:** Always include and configure `express-rate-limit` in the middleware chain *after* CORS.
